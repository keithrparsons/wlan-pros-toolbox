This release adds a convenience and fixes a consistency issue in signal grading.

NEW

The network tools now suggest the network you are actually on. Ping Sweep, Port Scan, and the ping tools start with your current subnet or gateway instead of a generic default. When the app can read your subnet mask, it fills in your true network. When it can only see your address, it assumes a /24 and says so, so you know to widen it if your network is larger. It never presents a guess as a measurement.

FIXED

Signal quality grading is now consistent across the app. The analyzer and the Signal Thresholds reference screen were grading on different dBm thresholds, so the same reading could be called "poor" on one screen and "fair" on another. Both now read from a single source, and a built-in check keeps them from ever drifting apart again. The thresholds are Excellent above -60 dBm, Good -60 to -67, Fair -67 to -72, Poor -73 and weaker.

The signal verdicts are also worded more honestly. A weak reading is guidance, not a verdict. Plenty of connections run fine at -75 dBm, and the app now says so instead of implying the number alone decides.
